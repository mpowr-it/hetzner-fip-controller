# Contributing Guide

Thank you for your interest in contributing to the Hetzner FIP Controller fork!

This document describes how to contribute code, documentation, and bug reports.

## Code Style

- Follow Go best practices and idiomatic Go code.
- Keep changes small and focused.
- Use meaningful commit messages (Conventional Commits are welcome).

## Workflow

1. Fork the repository
2. Create a new feature branch
3. Make your changes
4. Ensure the following:
    - `go build` passes
    - `go test` passes
    - CI pipeline is green
5. Open a Pull Request against `main`

## Reporting Issues

Please include:
- Expected behavior
- Actual behavior
- How to reproduce
- Logs if available
- Kubernetes version
- Hetzner Cloud environment details

## Code of Conduct

Be respectful, constructive, and helpful.  
Treat all contributors and maintainers with professionalism.
